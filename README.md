# Sanskrit Grammar Tutor Bot 🕉️

Адаптивный тренажер для изучения санскритской грамматики (субанты и тинанты) с использованием искусственного интеллекта. Проект автоматизирует процесс обучения, подстраиваясь под прогресс пользователя.

## 🌟 Основные возможности
- **Адаптивное обучение:** Система анализирует ваши ответы и фокусируется на 5 самых слабых темах, пока они не будут освоены (балл > 5).
- **Поддержка сложных форм:** Обработка склонений имен (Subanta) и спряжений глаголов (Tinanta).
- **Интеграция с LLM:** ИИ-учитель формулирует задания на естественном языке, строго соблюдая стандарты записи (Devanagari/IAST).
- **Надежное хранение данных:** Использование PostgreSQL с типами данных JSONB для хранения многомерных таблиц форм.

## 🌍 Мультиязычность (Bilingual Support)
Бот является полностью двуязычным и поддерживает **русский** и **английский** языки:
- **Автоматическое определение:** Бот определяет предпочтительный язык на основе настроек Telegram-профиля пользователя при первом контакте.
- **Гибкость:** Интерфейс и системные промпты ИИ переключаются динамически. Если вопрос задан на английском, ИИ-учитель переходит на английский, используя соответствующие термины (Case/Declension вместо Падеж/Склонение).
- **Локализация контента:** Все описания грамматических категорий в таблицах `subanta` и `tinanta` хранятся в формате JSONB с ключами `ru` и `en`, что позволяет мгновенно подтягивать нужный перевод.

## 🤖 Агентская архитектура (Agentic Workflow)
Проект построен на принципах **AI-агентов**, что позволяет системе не просто выдавать текст, а действовать как автономный субъект:
- **Разделение ролей:** Агент выступает в роли "Методиста" (выбирает данные из БД на основе прогресса) и "Учителя" (адаптирует эти данные в учебную форму под конкретного пользователя).
- **Инструментальное взаимодействие (Tool Use):** Агент не галлюцинирует формы санскрита, а использует строго определенные инструменты (Tools) для обращения к PostgreSQL. Это гарантирует 100% точность грамматических данных.
- **Целеполагание:** Вместо простого ответа на вопрос, агент преследует цель — довести уровень владения темой у пользователя до порога `passed` (6 баллов), самостоятельно принимая решение о смене темы или повторении пройденного.

## 🛠 Технологический стек
- **Language:** Python 3.10+
- **Database:** PostgreSQL (v15+)
- **ORM/DB Library:** Psycopg2 (Advanced JSONB operations)
- **AI Framework:** LangChain / OpenAI API
- **DevOps:** Docker, Docker-Compose
- **Environment:** Python-dotenv

## 📋 Архитектура базы данных
Проект использует три основные таблицы:
1. `subanta` / `tinanta` — метаданные тем и описание грамматических категорий.
2. `zabda` / `dhatu` — словари лемм и соответствующих им форм (JSONB).
3. `tg_users` — статистика прогресса, текущие баллы (`current_score`) и список пройденных тем (`passed`).

## 🚀 Быстрый старт

### 1. Клонирование репозитория

```bash
git clone [https://github.com/koleslena/sanskrit-trainer-bot.git](https://github.com/koleslena/sanskrit-trainer-bot.git)
cd sanskrit-trainer-bot
```

### 2. Настройка окружения

В корне проекта находится файл .env_example. Создайте на его основе свой локальный файл .env:

```bash
cp .env_example .env
```
Заполните в .env ваши учетные данные (API ключи и пароли к БД).

### 3. Запуск через Docker

Проект полностью контейнеризирован. Выполните команду:

```bash
docker-compose up -d --build
```

#### Что произойдет при первом запуске:

- Docker поднимет контейнер с PostgreSQL.

- Автоматически выполнится скрипт init.sql из директории /docker-entrypoint-initdb.d/.

- Будут созданы все таблицы (tg_users, subanta, tinanta, zabda, dhatu).

- База будет наполнена стартовыми данными (вставка всех основ и форм).


## 🧠 Логика обучения
Программа реализует алгоритм "Weakest Link":

- Если у пользователя менее 5 активных тем, бот предлагает новые случайные темы из базы, исключая уже пройденные (passed).

- Если активных тем 5 и более, бот выбирает тему с наименьшим количеством баллов для закрепления материала.

- За каждый правильный ответ начисляется +1 балл. При достижении 6 баллов тема считается выученной и переносится в архив.

## 📝 Формат ответа
Для корректной автоматической проверки ответов студент должен соблюдать следующие правила оформления:

1. **Единая строка:** Все формы (Sg, Du, Pl) вводятся в одну строку, разделенные только ОДНИМ пробелом.
2. **Звательный падеж (Sambodhana):** Частица `he` пишется раздельно со словом (например, `he deva`).
3. **Перифрастический перфект (CRITICAL):** Глагольные формы перифрастического перфекта (оканчивающиеся на `-ām`) **ОБЯЗАТЕЛЬНО** пишутся слитно со вспомогательным глаголом, без пробела.

Примеры: 

```
devaḥ devau devāḥ
```
```
he deva he devau he devāḥ
```
```
īkṣāñcakre īkṣāñcakrāte īkṣāñcakrire īkṣāñcakṛṣe īkṣāñcakrāthe īkṣāñcakṛḍhve īkṣāñcakre īkṣāñcakṛvahe īkṣāñcakṛmahe
```
